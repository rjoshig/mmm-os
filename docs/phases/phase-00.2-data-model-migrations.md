# Phase 0.2 — Data Model & Migrations

**Parent:** [`phase-00`](./phase-00-foundations-canonical-schema-data-model.md) ·
**Depends on:** 00.1 · **Status:** Not started

Covers Phase-0 requirement **P0-3** (and supports P0-5).

## Objective

Create the SQLAlchemy 2.0 ORM models for all Phase-0 tables and the Alembic
migration that builds them on an empty base — dialect-agnostic and multi-tenant
from day one.

## Scope

- **In:** ORM models for the Appendix C entities; portable column types; a single
  Alembic migration creating all Phase-0 tables.
- **Out:** business logic, services, API routes beyond health; tenant/user
  creation flow (00.3); config-versioning behaviour (00.3).

## Functional Requirements

- **P0.2-1** ORM models (typed `Mapped[...]`) for: `tenant`, `user`, `file`, `sheet`, `profile`, `mapping_config`, `taxonomy`, `taxonomy_alias`, `rule_set`, `rule`, `job`, `job_event`, `validation_flag`, `suggestion`, `output_row` ([`../data-model.md`](../data-model.md), Appendix C).
- **P0.2-2** Every domain table carries `tenant_id` (CC-1); `output_row` carries full traceability columns (CC-3): `source_file_id`, `source_sheet`, `source_row`, `mapping_config_version`, `rule_set_version`, `ingested_at`.
- **P0.2-3** Dialect-agnostic types only: `Uuid`/`String` PKs, `String` enums, `Numeric`, `DateTime(timezone=True)` (UTC), `JSON`. No Postgres-only types.
- **P0.2-4** `mapping_config` and `rule_set` carry an integer `version` column (config-as-data, CC-4).
- **P0.2-5** One Alembic autogenerated + reviewed migration creates all tables; `alembic upgrade head` and `downgrade base` both run cleanly on SQLite.

## Deliverables

- `src/mmm_os/models/*.py` — one module per entity (or grouped), all importing `Base`.
- Alembic version file under `migrations/versions/`.
- Tests: models import; `create_all`/migration produces the expected tables; every domain table has a `tenant_id` column.

## Acceptance Criteria

- `alembic upgrade head` creates all Phase-0 tables on an empty SQLite DB; `downgrade base` reverses cleanly.
- Every domain table has `tenant_id`; `output_row` has the traceability columns.
- Models type-check under mypy strict; no Postgres-only types used.

## Dependencies
